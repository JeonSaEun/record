* Reduction (리덕션)
- 대량의 데이터를 가공해 축소

* 람다식 -> 메소드 참조로 변환할 수 있는 조건
(1) 람다식의 매개변수가 호출메소드의 매개변수로 단순 전달하는 경우
(2) 람다식의 매개변수가 가지고 있는 메소드를 단순 호출하는 경우
(3) 람다식의 실행블록의 내용이 단순 1개의 객체를 생성하는 코드일 경우(="생성자 참조")

* 람다식 -> 메소드/생성자 참조로 바꾸는 절차
(1) 매개변수 삭제
(2) 람다식 기호 삭제(->)
(3) 람다식 실행문장의 호출메소드에 전달된 매개변수도 삭제
(4) 람다식의 실행 메소드가 3가지 경우가 존재할 수 있음
	1. 정적 메소드 호출 		(메소드참조)
	: 클래스명::정적메소드명		 		=> ParallelExample::print
	2. 인스턴스 메소드 호출 	(메소드참조)
	: 참조변수명::인스턴스메소드명			=> System.out::println
	3. 람다식의 매개변수가 가지고 있는 메소드를 단순 호출 (메소드참조)
	: 매개변수의 타입명::인스턴스메소드명	=> String::length
	4. new 생성자() 호출 	(생성자참조)
	: 생성할 클래스명::new				=> String::new

* 스트림 파이프라인의 수행은 지연실행(Lazy Execution)기반으로 동작
	: 지연실행이란? 중간처리메소드의 수행을 바로 수행시키지 않고,
	  최종처리 메소드가 수행될 때, 한꺼번에 중간처리메소드를 수행시킴.
